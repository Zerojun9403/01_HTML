form태그 / input태그

form태그

        - 입력양식을 작성하는 판(영역, 공간)
        - 내부에 작성된 입력 값을 지정된 곳으로 제출하는 역할

        [속성]
        1)action 제출할 곳을 지정하는 속성
        2)mathod(방식) : 설정된 곳으로 데이터를 전달하는 방식
            - 서버에 저장할것인지,서버에 저장된 데잍터를 호출할것인지 방식선택
        3)name : 이름을 지정하는 속성(JS 사용)
        4)target : 제출된 페이지를 현재/새 탭으로 열기

input태그

        웹 문서에서 사용자(클라이언트)가 입력 할 수 있는 양식을 제공하는 태그 
        많은 타입이 존재하고, input 의 변형 형태도 다수 존재

        * input 태그에는 name 속성이 필수로 작성되어야 한다
        (꼭은 아니지만 작성하는 경우가 80% 이상 많음)

날짜/시간 관련 input 태그

radio, cheek box

      -여러 연관된 값을 묶어서 선택하는 경우에 사용 
      - radio : 여러 값중 하나 선택
        ex) 성별
      - cheekbox: 여러값중원하는 모든 것 선택
        ex) 상세검색, 관심분야, 취미

        
          연관된 radio,cheekbox는 같은 name 속성 값을 가져야 한다.

radio확인

A팀 B팀 C팀 D팀

cheekbox 확인

RED YELLOW ORANGE GRREN

label 태그

      - 입력관련 태그(input,select, texarea 등)의 제목을 나타내는 태그

      -사용법 1 : label 태그 내부에 입력 관련된 태그 하나 작성하면 label 태그를 클릭해도 입력 관련 태그가 선택된다. 

      -사용법 2:  입력 관련 태그의 id 속성값을 
                label 태그 의 for 속성값에 똑같이 적으면 연결된다.

select / opton

      select : 클릭하면 밑으로 메뉴가 내려오는 입력 태그(드롭 다운)

      -opton : select 클릭시 보여질 메뉴를 작성하는 태그

      *select 와 option은 input과 name, value 가 분리된 태그로 생각하면 된다.
초밥 샐러드 라면 햄버거




textarea

      textarea글자 지역
      - 여러줄을 입력할 수 있는 입력 상자
      [속성]
      -name : 재출되는 곳에서 데이터를 읽을때 필요한 이름
      -rows : textarea의 행 크기를 지정(row="5" -> 5행)
      -cols : textarea의 행 크기를 지정(cols="5"-> 5열)

버튼 태그

      - 버튼을 만드는 태그 

      - type : 속성값에 따라서 동작이 달라짐

      type="submit" 제출 버튼을 만들때 사용 
      (button 태그의 기본값 == type 속성을 작성하지 않으면 자동 sumit 설정)

      type="reset" : 버튼이 포함된 form 태그의 모든 입력 값을 초기화

      type ="button" : 아무런 기능이 없는 버튼
                  -> 제출 이외에 개발자가 원하는 기능으로 커스텀해서 사용할 때 사용
                      javaScript에서 기능작성하고 추가하는 용도의 버튼
